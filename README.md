# Проблема: 
Отсутствует универсальная и масштабируемая платформа, которая
позволяет централизованно управлять каталогом мини-приложений (miniapp),
предоставлять к ним прозрачный доступ через административную панель, а также
легко и безопасно встраивать их во внешние приложения (мобильные и веб) с
обеспечением бесшовной (single sign-on) авторизации пользователей.


# В текущей архитектуре мы решили следующие проблемы:
● Нет единого места для хранения и публикации миниаппов;
● Сложно централизованно управлять доступами, ролями и настройками
миниаппов для разных пользователей и групп;
● Отсутствует возможность мгновенного подключения миниаппов к внешним
приложениям без сложной ручной интеграции;
● Нет механизма бесшовной авторизации — пользователи сталкиваются с
повторным вводом логина/пароля при каждом переходе между сервисами, что
ухудшает пользовательский опыт и снижает безопасность;
● Внешние приложения требуют доработки под каждый новый миниапп, что
затрудняет расширение экосистемы и увеличивает стоимость поддержки;
● Отсутствует эффективная система мониторинга, сбора статистики и аудита
действий пользователей/администраторов.
